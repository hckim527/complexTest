## 템플릿 구조 안내

### 완성된 화면 템플릿 (page/)

일반적으로 다음과 같은 카테고리로 구성되어 있습니다:

**단일 화면 (Single/Basic)**:
- 입력폼, 상세조회, 로그인 등 단일 목적 화면
- 예: `simple_form.xml`, `detail_view.xml`, `login.xml` 등

**복합 화면 (Multi/Composite)**:
- 검색+그리드, 마스터-디테일, 탭 등 여러 요소 조합
- 예: `search_grid.xml`, `master_detail.xml`, `tab_container.xml` 등

### 코드 조각 (snippets/)

화면의 일부 기능을 추가할 때 참고할 수 있는 코드 조각입니다:

- 화면 레이아웃 (화면시작, 화면분할 등)
- UI 컴포넌트 (타이틀, 검색영역, 그리드, 버튼 등)
- 특수 기능 (탭, 트리, 아코디언 등)

**참고**: 프로젝트마다 디렉토리명과 파일명이 다를 수 있습니다. 위의 실제 구조를 확인하세요.

## 템플릿 활용 방법

### 새 화면을 만들 때

**방법 1: 템플릿 기반 (빠른 구현)**
```
1. 요구사항과 유사한 템플릿 찾기
2. 템플릿 xml 파일을 Read 도구로 읽어서 구조 파악
3. DataCollection, 레이아웃, 스크립트 패턴 확인
4. 필요한 부분을 수정하여 사용
```

**방법 2: 처음부터 작성 (유연한 구현)**
```
1. Global 가이드의 기본 구조 참고
2. Global 가이드를 참고해서 필요한 컴포넌트 스펙 확인
3. 요구사항에 맞게 자유롭게 작성
```

**어떤 방법을 선택할지**:
- 프로젝트에 표준 패턴이 있고, 그것을 따라야 한다면 → 방법 1
- 기존 패턴과 다른 새로운 화면이라면 → 방법 2
- 둘의 조합도 가능

### 기존 화면을 수정할 때

**기본 원칙**: 기존 화면의 패턴과 구조를 유지하는 것이 우선입니다.

```
1. 기존 화면 코드를 읽고 현재 구조 파악
2. 수정이 필요한 부분만 변경
3. 새 기능 추가 시에는 snippets 참고 가능
```

### 일부 기능만 추가할 때

```
1. snippets/ 폴더에서 해당 기능 찾기
2. xml 파일을 읽어서 패턴 확인
3. 기존 화면에 통합 (기존 스타일과 조화롭게)
```

예시:
- 검색 조건 추가 → `snippets/03_조회영역/` 또는 `search/` 참고
- 버튼 추가 → `snippets/08_버튼/` 또는 `button/` 참고
- 그리드 추가 → `snippets/06_그리드/` 또는 `grid/` 참고

## 템플릿 읽기 팁

### 파일을 직접 읽기

디렉토리 목록만으로는 내용을 알 수 없습니다. Read 도구로 xml 파일의 내용을 읽어서 다음을 확인하세요:

- **레이아웃 패턴**: 어떤 CSS 클래스를 사용하는가?
- **스크립트 패턴**: 함수명은 어떻게 짓는가? 어떤 구조인가?
- **컴포넌트 설정**: 실제로 어떤 속성들을 사용하는가?

### 파일명은 참고만

프로젝트마다 파일명이 다를 수 있습니다:
- `search_grid.xml`이 없다면 `list.xml`, `search_list.xml` 등을 찾아보세요
- 정확한 이름보다는 **파일 내용**이 요구사항과 맞는지 확인하세요
- 여러 파일을 읽어보고 가장 적합한 것을 선택하세요

## 작업 가이드

### 새 화면 작성

```
Step 1: 요구사항 분석
  - 어떤 기능이 필요한가?
  - 유사한 기존 화면이 있는가?

Step 2: 접근 방법 결정
  - 템플릿 활용할 것인가?
  - 처음부터 만들 것인가?
  - 기존 화면을 복사할 것인가?

Step 3: 구현
  - Global 가이드의 기본 규칙 준수
  - 템플릿 또는 기존 패턴 참고
  - 컴포넌트 스펙 확인

Step 4: 검증
  - XML 구조가 올바른가?
  - 프로젝트 패턴과 일관성이 있는가?
```

### 기존 화면 수정

```
Step 1: 현재 코드 이해
  - 기존 화면 읽기
  - 현재 구조와 패턴 파악

Step 2: 수정 범위 결정
  - 어디를 어떻게 수정할 것인가?
  - 기존 패턴을 유지할 수 있는가?

Step 3: 구현
  - 기존 스타일과 구조 유지
  - 필요한 부분만 변경
  - snippets 참고 (필요 시)

Step 4: 일관성 확인
  - 기존 코드와 조화로운가?
  - 다른 부분에 영향은 없는가?
```

# complexTest — Jira 결함 테스트 샘플 검증 규칙

> **워크플로우: CT_01_생성 → CT_02_검증 → CT_03_보강**
> - `complexTest/CT_01_생성.md` 로 생성된 샘플을 대상으로 검증한다.
> - 검증 완료 후 `complexTest/CT_03_보강.md` 규칙으로 보강한다.

---

## 1. 검증 항목

1. **`validate_xml` (필수)**
   - WebSquare XML 형식(네임스페이스, head/body 구조)이 올바른지 확인.
   - `description`(Jira 제목), `validation`(회귀 케이스)이 정의되어 있는지 확인.

2. **MCP playwright 렌더 검증 (필수)**
   - wpack 자동 컴파일 후, 포트 자동감지하여 미리보기 wrapper(`websquare.html?w2xPath=`)로 페이지를 연다.
   - 확인 범위:
     1) 페이지가 정상 로딩되고 첨부 컴포넌트(`target1` 등)가 생성되는지 (`window.mf_target1` 존재 여부)
     2) 재현/검증 핸들러 호출 후 `wf_body_bottom` 결과창에 값이 정상 표시되는지
     3) console error 0
   - 페이지 scope 적용으로 `target1` → `mf_target1` 자동 prefix → 외부 evaluate 시 `window.mf_target1` 사용.

3. 결과 보고에 "검증: PASS / 발견 이슈" 를 포함한다.

---

## 2. 검증 절차

1. `validate_xml` 로 구조를 확인하고 오류를 수정한다.
2. MCP playwright 로 렌더/핸들러 동작/console error 0 을 확인한다.
3. 오류 발견 시 샘플 XML 을 수정하고 재검증한다.

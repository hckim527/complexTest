# complexTest — Jira 결함 재현 샘플 규칙 (CT_04)

> 기본 규칙은 `CT_01_생성.md`·`CT_AGENT_RULES.md` 를 따르고, 본 문서는 **Jira 결함 재현/연계 특화** 사항만 둔다.
> (유실 후 재작성 — 상세 규칙은 샘플을 생성해가며 보강한다.)

---

## 1. 대상
- `[QA]` 프리픽스를 **제외**한 Jira 결함 중 **복합조건** 건.
- **Jira 첨부 샘플의 컴포넌트를 그대로 사용**한다(검증된 구성으로 렌더/재현 정확).

## 2. 파일/경로
- 경로: `complexTest/src/main/webapp/sample/{컴포넌트}/`
- 파일명: `WAEA_{번호}_{컴포넌트}_{기능}_1.xml`
- 첨부 다운로드: `workspace/_jira_attachments/{KEY}/` (**webapp 밖** — Studio 컴파일/서빙 대상 제외; `src/main/webapp/` 안에 두지 말 것). 다운로드는 API 토큰 basic-auth(`reference_jira_attachment_api_token`).

## 3. 구성 (재현 + 대조군)
- **결함 조건 컴포넌트**와 **대조군**(조건 미적용 정상 케이스)을 나란히 배치해, 결함이 해당 조건 때문임을 입증한다.
- **재현 버튼**: 사용자가 결함을 재현·측정할 수 있는 실행 버튼을 제공하고, 결과(값/DOM 상태/수치)를 `wf_body_bottom.getObj("scwin").setReturnValue(...)` 로 출력한다.
- `description` = Jira 결함 제목 원문 + `<br/>` + Jira 링크(`{KEY}`, 파란 밑줄, 새 탭). 형식은 CT_01 참조.
- `validation` = **등록 현상 회귀 + 현상 외 추가 검증 케이스**(complexTest 목적: 테스트 커버리지 향상).
- **금지**: 결과창에 `[재현 절차]` 텍스트 출력(절차/전제/원인은 `createLabel` 라벨로). 화면 표시 정보·코드 주석에 엔진 js 파일명·라인·내부 메서드명.

## 4. 검증
- `validate_xml`(필수) + MCP 렌더/동작/`console error 0` 확인.
